---
title: Authentication Overview
description: Claudio's authentication options.
---

Claudio supports multiple authentication methods that can be used together:

- **Local accounts** — Username/password registration and login
- **GitHub OAuth** — Sign in with GitHub
- **Google OAuth** — Sign in with Google
- **Custom OIDC** — Any OpenID Connect provider (Authentik, Authelia, Pocket ID, etc.)
- **Proxy authentication** — Trust a header from a reverse proxy for SSO

## Session Model

- **Web** uses ASP.NET Identity cookie sessions. The browser does not store JWT access tokens or refresh tokens.
- **Desktop** uses Claudio-issued bearer access tokens plus opaque refresh tokens stored in secure local storage.
- **Web auth state** is loaded from `GET /api/auth/me`.

If the browser app and API are served from different origins, set `CLAUDIO_BROWSER_ORIGINS` to the browser-visible frontend origin or origins. For same-origin reverse-proxy deployments, leave it unset.

## First User

The first user created (local or external) automatically becomes admin.

## Account Linking

When a user signs in with an external provider whose verified email matches an existing account, the accounts are automatically linked.

## Disabling Auth

| Variable                             | Effect                                                           |
| ------------------------------------ | ---------------------------------------------------------------- |
| `CLAUDIO_DISABLE_AUTH=true`          | Disable authentication entirely — open access, everyone is admin |
| `CLAUDIO_DISABLE_LOCAL_LOGIN=true`   | Disable username/password login, require external providers only |
| `CLAUDIO_DISABLE_USER_CREATION=true` | Block new account creation. Existing users can still sign in     |
