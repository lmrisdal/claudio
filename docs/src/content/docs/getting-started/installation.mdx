## Docker

```bash
docker run -d \
--name claudio \
-e PUID=1000 -e PGID=1000 \
-p 8080:8080 \
-v /path/to/games:/games \
-v claudio-data:/config \
ghcr.io/lmrisdal/claudio:latest
```

Open `http://localhost:8080` and register your first user (automatically gets admin role).

## Docker Compose

```yaml
services:
  claudio:
    image: ghcr.io/lmrisdal/claudio:latest
    container_name: claudio
    ports:
      - "8080:8080"
    volumes:
      - claudio-data:/config
      - /path/to/games:/games
    environment:
      - PUID=1000
      - PGID=1000
      - CLAUDIO_IGDB_CLIENT_ID=your_client_id
      - CLAUDIO_IGDB_CLIENT_SECRET=your_client_secret
      # Same-origin reverse proxy: leave CLAUDIO_BROWSER_ORIGINS unset.
      # Multiple origins should be comma-separated with no spaces.
      # - CLAUDIO_BROWSER_ORIGINS=https://claudio.example.com,https://staging.example.com

volumes:
  claudio-data:
```

## Docker Compose with PostgreSQL

```yaml
services:
  claudio:
    image: ghcr.io/lmrisdal/claudio:latest
    container_name: claudio
    ports:
      - "8080:8080"
    volumes:
      - claudio-data:/config
      - /path/to/games:/games
    environment:
      - PUID=1000
      - PGID=1000
      - CLAUDIO_DB_PROVIDER=postgres
      - CLAUDIO_DB_POSTGRES=postgres://claudio:secret@db/claudio
      # Same-origin reverse proxy: leave CLAUDIO_BROWSER_ORIGINS unset.
      # Multiple origins should be comma-separated with no spaces.
      # - CLAUDIO_BROWSER_ORIGINS=https://claudio.example.com,https://staging.example.com
    depends_on:
      - db

  db:
    image: postgres:18-alpine
    container_name: claudio-db
    volumes:
      - postgres-data:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=claudio
      - POSTGRES_USER=claudio
      - POSTGRES_PASSWORD=secret

volumes:
  claudio-data:
  postgres-data:
```

## Data Persistence

Claudio stores a signing key (`claudio-signing.key`) alongside the database in the config directory. This key signs authentication tokens — if it's lost, all users will need to sign in again.

Make sure the `/config` volume is persistent.

## Reverse Proxy And Browser Origins

The browser app now uses cookie sessions. If your reverse proxy serves the SPA and API from the same origin, such as `https://claudio.example.com` and `https://claudio.example.com/api`, leave `CLAUDIO_BROWSER_ORIGINS` unset.

If the SPA and API are served from different origins, set `CLAUDIO_BROWSER_ORIGINS` to the browser-visible frontend origin. For multiple frontend origins, use a comma-separated list with no spaces.

This is an advanced setup. The normal Docker and reverse-proxy deployment path is same-origin, so most installations do not need this setting.
